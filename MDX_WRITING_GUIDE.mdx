# MDX Writing Guide

This guide shows you how to write MDX files with all the formatting options available.

## Frontmatter

Every MDX file should start with frontmatter between triple dashes:

```yaml
---
title: "Your Post Title"
date: "2024-01-15"
excerpt: "A brief description of your post"
author: "Your Name"
tags: ["tag1", "tag2", "tag3"]
image: "/images/featured-image.jpg"  # optional
published: true  # optional
---
```

## Text Formatting

You can format text in several ways:

- **Bold text** using double asterisks or __double underscores__
- *Italic text* using single asterisks or _single underscores_
- ***Bold and italic*** using triple asterisks
- ~~Strikethrough~~ using double tildes
- `Inline code` using backticks

## HTML in MDX

You can use HTML directly in MDX:

<div style={{backgroundColor: '#f0f0f0', padding: '20px', borderRadius: '8px'}}>
  <h3>This is HTML</h3>
  <p>You can use HTML tags and inline styles.</p>
</div>

## React Components in MDX

You can import and use React components:

```mdx
import { Button } from '@/components/Button'
import { Alert } from '@/components/Alert'

<Button onClick={() => alert('Clicked!')}>
  Click me
</Button>

<Alert type="info">
  This is an info alert component
</Alert>
```

## JSX Expressions

You can use JSX expressions with curly braces:

The current year is {new Date().getFullYear()}.

2 + 2 = {2 + 2}

## Custom Components

You can create custom components within MDX:

export const Highlight = ({children, color}) => (
  <span style={{backgroundColor: color || 'yellow', padding: '2px 6px', borderRadius: '4px'}}>
    {children}
  </span>
);

Now use it: <Highlight color="lightblue">This text is highlighted</Highlight>

## Escaping

To show literal characters that would normally be formatting, use backslash:

- \*This won't be italic\*
- \`This won't be code\`
- \# This won't be a heading

## Best Practices

1. **Start with frontmatter** - Always include metadata
2. **Use meaningful headings** - Structure your content logically
3. **Break up long content** - Use headings, lists, and paragraphs
4. **Add alt text to images** - For accessibility
5. **Use code blocks for code** - With appropriate language highlighting
6. **Preview your content** - Check how it renders before publishing

## Ending an MDX File

MDX files don't require special ending syntax. Just make sure:
- All opened tags are closed
- All code blocks are properly closed with ```
- The content ends naturally
- No trailing spaces or unnecessary blank lines
